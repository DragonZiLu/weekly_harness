# H30269 红利低波50 复现 — 完整 Debug 报告

> 日期：2026-06-14 | 目标：尽可能准确复现中证红利低波指数 H30269

---

## 一、实验背景

复现中证红利低波50指数 H30269，使用中证800作为样本空间（用户指定），与官方全收益指数 H20269 对标。

### 官方编制方案核心规则

1. **样本空间**：中证全指中满足：连续3年税后分红>0 + 日均市值前80% + 日均成交额前80%
2. **选样**：剔除支付率过高(前5%)/为负 + 剔除DPS增长率非正 → 三年平均税后股息率 Top75 → 波动率 Top50
3. **加权**：股息率加权，单只≤15%
4. **调仓**：每年12月第二个周五下一交易日
5. **缓冲区**：每次调整≤20%（保留≥40只原样本）

## 二、Debug 三轮迭代

### v1：逐年版（初始实现）

| 实现 | 说明 |
|------|------|
| DPS增长 | 每年对前一年增长>0（4年数据，3个增长率全部>0） |
| 缓冲区 | ❌ 未实现 |
| 样本空间 | CSI 800 |

**结果**：重合度 30%，年化 7.05% vs 官方 8.65%（落后1.60pp）

**问题**：DPS"每年都正增长"淘汰了 87.8% 分红合格样本，周期行业（煤炭/钢铁/化工）全军覆没。

### v2：+20% 缓冲区

**改动**：新增 `_apply_rebalance_buffer`，确保每期至少40只原样本保留

**结果**：重合度 30→34.5%，换手率 48-64%→20%，年化微升至 7.12%

**问题**：缓冲区补回的原样本都在 DPS 过滤后的池子外，补回效果有限。

### v3：DPS 改为解读B + 缓冲区 ✅

**改动**：DPS 增长过滤从「每年都正增长」改为「三年累计增长>0」

```python
# 解读A（逐年版）❌
for i in range(1, 4):
    if dps[i] / dps[i-1] - 1 <= 0:  # 每年都要增长
        return False

# 解读B（累计版）✅
growth = dps[3] / dps[0] - 1      # 只要最终比三年前高
return growth > 0
```

**效果**：DPS 通过数从 71→270（+3.8x），茅台、神华、紫金等周期分红股终于能进入候选池。

## 四、关键发现

### 4.1 解读B 更符合官方意图

官方原文「过去三年的每股股利增长率」是**单一累计指标**，与「过去一年的红利支付率」并列，而非每年都要单独检查。语法和逻辑上都支持解读B。

### 4.2 官方成分股不完全在 CSI 800 内

约 18.4% 的官方 H30269 成分股不在 CSI 800（中证800）样本空间内，属于中盘股（如 600674 川投能源、002032 苏泊尔等）。这是剩余重合度差距的主因。

### 4.3 缓冲区是复现的必要条件

没有 20% 缓冲区时换手率高达 46-64%，加了之后稳定在 20%，与官方规则一致。

### 4.4 仍有残留差异

重合度 47.8% vs 理论上限 ~82%，剩余差距可能来自：
- 样本空间差异（中证全指 vs CSI 800）
- 股息率计算口径（各年平均 vs ΣDPS/股价）
- 流动性过滤（市值/成交额前80%）
- 支付率计算方式（单年 vs 多年）

## 五、结论

**解读B（三年累计增长>0）+ 20%缓冲区** 是本次 debug 的核心突破。

- **v3 版本年化 8.95% 微幅跑赢官方全收益 8.65%（+0.30pp）**
- **相关系数 0.956，跟踪误差 3.87%，持仓重合度 47.8%**
- **可作为 H30269 的有效近似复现**
- 如需更高重合度，可进一步扩大样本空间至中证全指
