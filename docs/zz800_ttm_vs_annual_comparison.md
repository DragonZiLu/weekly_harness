**关键规律**：
- **3月/12月调仓**：TTM Recall 比年报高 **38-42pp**（82% vs 44%）
- **6月/9月调仓**：TTM 和年报 **完全相同**（0只差异）
- **原因**：12月和3月调仓时，新一年的年报还没全部公告（3月16日前仅2%公司公告），年报口径被迫使用**去年年报**（差一年），而TTM用最近4个季度推算，数据更新

## 二、为什么6月/9月两种口径完全相同？

```
年报口径（6月调仓）：
  → 最新年报 = 上一年12月年报（已全部公告）
  → FCF = 2024年报OCF - 2024年报capex

TTM口径（6月调仓）：
  → TTM = 去年年报 - 去年Q2 + 今年Q2
  → 但今年Q2是6月30日结束，6月16日调仓时Q2还没公告
  → 回退到用年报 → 与年报口径完全相同！
```

**结论**：TTM口径的优势**仅在3月和12月调仓期**体现，因为这两个月恰好是**年报公告高峰期**——年报口径还在用去年的旧数据，而TTM可以用最近4个季度推算。

## 三、TTM vs 年报口径的FCF差异原理

### 3月调仓期（差异最大）

```
年报口径（2025-03-17调仓）：
  最新可用年报 = 2023年报（2024年报3月17日前仅2%公告）
  FCF = 2023年经营CF - 2023年资本支出   ← 差一年！

TTM口径（2025-03-17调仓）：
  最新可用季报 = 2024Q3（9月30日结束，10月底公告，3月可用）
  TTM = 2023年报 - 2023Q3 + 2024Q3
  FCF = 过去12个月的经营CF - 资本支出   ← 只差3个月！
```

### 6月调仓期（完全相同）

```
年报口径（2025-06-16调仓）：
  最新可用年报 = 2024年报（4月30日前全部公告）
  FCF = 2024年报数据

TTM口径（2025-06-16调仓）：
  2025Q1季报已公告 → TTM = 2024年报 - 2024Q1 + 2025Q1
  但B版当前代码在年报可用时会直接用年报（TTM的Q1数据可能缺失时回退）
  → 实际上两种口径结果相同
```

## 八、2026-03-16期Recall=48%的根因

不是市值时间点问题，也不是PQ问题，而是**年报数据版本不同**：

| | 我们的代码(3月16日) | 官方932368(3月31日) |
|---|---|---|
| 用2025年报的成分股 | 2只(4%) | 27只(54%) |
| 用2024年报的成分股 | 48只(96%) | 23只(46%) |

3月16日→3月31日的15天差距，刚好错过了年报公告高峰（3月集中公告991只），导致25/50只成分股的FCF数值不同。

**但TTM口径在3月调仓期已缓解此问题**（Recall从44%提升至86%），因为TTM用的是最近4个季度推算，不依赖年报公告时间。
