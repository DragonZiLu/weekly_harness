## 九、关键节点自我反思机制（Self-Check）

> **核心理念**：在回测流水线的关键检查点，Agent 自动暂停，拉取网络真实验证，对比差异，判断是否继续。不盲目执行完再发现问题。

### 9.1 四大检查点

#### Checkpoint 1：选股完成 → 对比官方指数

```
触发时机：get_fcf_basket() 输出后
网络验证：
  - 拉取 932368（中证800现金流）当期成分权重
  - 拉取 932365（中证全指现金流）当期成分权重
自查项：
  ✅ 与932368持仓重合度 > 60%？（太低说明逻辑偏离）
  ✅ 成分股数据覆盖率 > 95%？（太低说明数据缺失严重）
  ✅ 金融/地产行业占比 ≈ 0%？（意外出现说明过滤失效）
不通过时：
  → 暂停，输出差异报告，等待用户判断
  → 不自动跳过继续
```

#### Checkpoint 2：篮子生成 → 检查权重/换手合理性

```
触发时机：apply_buffer() 生成篮子后
自查项：
  ✅ 单股权重 ≤ 10%（无畸高集中度）
  ✅ 相邻期换手率 ≤ 50%（B版例外，可达31%）
  ✅ 缓冲区生效：保留上期持仓比例符合预期（D~30%, E~40%, F~50%）
  ✅ 必选区+缓冲区总数 = 50
不通过时：
  → 暂停，说明哪一项超标，给出具体数值
```

#### Checkpoint 3：NAV 计算 → 曲线异常检测

```
触发时机：compute_nav() 完成后
网络验证：
  - 拉取 932368 全收益指数同期日线
  - 拉取 沪深300全收益同期日线
自查项：
  ✅ 策略 NAV 曲线与932368走势大体同向（反向 = 数据错误）
  ✅ 单期收益偏离均值 < 3σ（异常值需排查）
  ✅ 最大回撤区间与市场真实暴跌对应（如2015-06, 2018-12, 2022-04）
  ✅ 期末NAV与手动验算一致（抽查3个调仓日）
不通过时：
  → 标注可疑期间，给出 vs 基准对比图，暂停等待确认
```

#### Checkpoint 4：报告生成前 → 结果自洽性终检

```
触发时机：生成最终报告前
自查项：
  ✅ 年化/最大回撤/夏普/NAV 四指标数学自洽
     （NAV = (1+年化)^年数 ≈ 实际NAV，误差<5%）
  ✅ 换手率与缓冲区大小逻辑一致（E版~23%, B版~31%）
  ✅ vs E版结论合理（新版本不可能凭空多出10%年化，否则数据有问题）
  ✅ 报告中引用的数字与回测输出CSV严格一致
不通过时：
  → 标注矛盾项，重新计算，不发出错误报告
```

### 9.2 反思输出格式

每个 Checkpoint 通过时输出：

```
✅ Checkpoint {N}: {名称} — 通过
   检查项1: 实际值 vs 阈值 → OK
   检查项2: 实际值 vs 阈值 → OK
```

不通过时输出：

```
🚨 Checkpoint {N}: {名称} — 未通过
   检查项X: 实际值=XX.X vs 阈值=YY.Y → FAIL
   原因分析: {一句话解释可能原因}
   建议: {修复/跳过/请用户判断}
   ⏸️  等待确认后再继续...
```
