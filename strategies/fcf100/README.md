# 中证全指自由现金流指数策略

> 基于中证指数公司 **中证全指自由现金流指数（932365）** 编制方案的回测实现

## 一、策略概述

本策略严格遵循 [中证全指自由现金流指数编制方案 V1.0（2024年12月）](https://www.csindex.com.cn)，从中证全指样本空间中选取 **100 只自由现金流率最高的上市公司证券**，按自由现金流加权（单样本上限 10%），每季度调仓一次。

### 核心逻辑

自由现金流（Free Cash Flow, FCF）是企业维持运营和增长后可自由分配给股东和债权人的现金，是衡量企业真正"赚钱能力"的黄金指标。与净利润不同，FCF 难以被会计手段操纵，更能反映企业的真实盈利质量和财务健康度。

**自由现金流率 = 自由现金流 / 企业价值**，类似于股票的"现金回报率"，数值越高代表市场对其现金创造能力的定价越便宜。

## 二、指数编制方案（对齐官方 V1.0）

### 基本信息

| 项目 | 内容 |
|:---|:---|
| 指数名称 | 中证全指自由现金流指数 |
| 指数简称 | 中证现金流 |
| 指数代码 | 932365 |
| 基日 | 2013年12月31日 |
| 基点 | 1000 点 |

### 样本选取

#### Step 1: 样本空间
同中证全指指数样本空间（000985.SH）

#### Step 2: 可投资性筛选
过去一年日均成交金额排名位于样本空间前 80%

> 实现方式：通过 `daily_basic` 获取调仓日附近交易日的换手率（`turnover_rate_f`）和流通市值（`circ_mv`），
> 估算日均成交额 = 换手率(%) × 流通市值 / 100，排序取前 80%。

#### Step 3: 选样方法

**（1）行业剔除**
- 按中证一级行业分类，剔除**金融**和**房地产**行业
- 原因：金融和房地产行业的现金流模式独特，与其他行业差异较大

**（2）财务质量过滤**
| 条件 | 公式 | 说明 |
|:---|:---|:---|
| FCF > 0 | 经营活动现金流量净额 - 购建固定资产等支付的现金 | 必须有正向自由现金流 |
| EV > 0 | 总市值 + 总负债 - 货币资金 | 企业价值必须为正 |
| 5年经营CF为正 | 连续5个完整会计年度经营活动现金流量净额 > 0 | 现金流持续性验证 |
| 盈利质量前80% | (经营CF - 营业利润) / 总资产 | 剔除盈利质量最差的20% |

**（3）排名选取**
- 按自由现金流率 = FCF / EV 从高到低排名
- 选取前 **100 只**证券作为指数样本

### 加权方式

- **自由现金流加权**：权重 ∝ 样本的自由现金流绝对值
- **单样本权重上限**：10%
- 权重因子在每季度定期调整时更新

### 调仓频率

- **定期调整**：每季度一次
- **调整时间**：每年 3月、6月、9月、12月的**第二个星期五的下一交易日**
- 权重因子随样本定期调整而调整

## 三、回测实现架构

```
weekly_harness/
├── fcf_universe.py     ← FCF 选股核心（财务数据缓存 / FCF 计算 / 筛选逻辑）
├── backtest.py          ← 回测引擎（新增 fcf100 宇宙支持 + FCF 调仓日）
│
backtest_fcf.py          ← FCF 策略回测入口
strategies/fcf100/
└── README.md            ← 本文档
```

### 关键模块

| 模块 | 说明 |
|:---|:---|
| `FinancialDataCache` | 批量下载/缓存 cashflow、balancesheet、income 全市场数据 |
| `IndexWeightCache` | CSI 全指（000985.SH）成分股权重缓存 |
| `FcfUniverse.get_fcf_basket()` | 核心选股逻辑：成分股 → 行业剔除 → FCF 计算 → 质量过滤 → Top N |
| `BacktestEngine` (fcf100模式) | FCF 加权权重直接作为目标仓位，绕过红利评分体系 |

### 数据流

```
tushare API
    │
    ├── cashflow (n_cashflow_act, c_pay_acq_const_fiolta)
    ├── balancesheet (total_liab, money_cap, total_assets)
    ├── income (oper_profit)
    ├── daily_basic (total_mv)
    └── index_weight (000985.SH 成分股)
            │
            ▼
    data/fcf_financials/*.csv   ← 本地缓存
            │
            ▼
    FcfUniverse.get_fcf_basket(date_str)
            │
            ├── FCF = 经营CF - 资本支出
            ├── EV = 总市值 + 总负债 - 货币资金
            ├── FCF率 = FCF / EV
            ├── 盈利质量 = (经营CF - 营业利润) / 总资产
            │
            ▼
    Top 100 by FCF率, FCF加权(≤10%)
```

### 防前视偏差（Look-ahead Bias）保障

1. **年报数据**：仅使用调仓日之前已公告的最新完整年报
   - 5月之前 → 用前年年报（当年年报通常在 4 月底才公告完毕）
   - 5月及之后 → 用上年年报
2. **成分股**：使用调仓日之前最近可用的指数权重快照（前向填充）
3. **市值数据**：使用调仓日当日或最近的前一交易日 daily_basic
4. **连续5年OCF**：只检查调仓日时已知的已公告年报数据

## 四、使用方法

### 首次运行（下载数据）

```bash
# 下载全市场 2010-2026 年财务数据（约需 10-15 分钟）
python backtest_fcf.py --start 2020-01-01 --download
```

### 回测

```bash
# 2020-2026 年回测
python backtest_fcf.py --start 2020-01-01

# 2015 年至今完整回测
python backtest_fcf.py --start 2015-01-01

# 选取前50只（更集中）
python backtest_fcf.py --top-n 50 --start 2020-01-01

# 指定基准对比
python backtest_fcf.py --start 2020-01-01 --benchmark 000300.SH,000985.SH

# 低换手模式（调仓阈值5%）
python backtest_fcf.py --start 2020-01-01 --rebalance-threshold 5.0
```

### 参数说明

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| `--start` | 2020-01-01 | 回测开始日期 |
| `--end` | 2026-06-01 | 回测结束日期 |
| `--cash` | 1,000,000 | 初始资金（元） |
| `--top-n` | 100 | 选取前 N 只标的 |
| `--max-weight` | 10.0 | 单股最大权重% |
| `--rebalance-threshold` | 2.0 | 调仓触发阈值% |
| `--commission` | 0.1 | 佣金率‱ |
| `--slippage` | 0.1 | 滑点‱ |
| `--benchmark` | 000300.SH | 基准指数代码 |
| `--download` | False | 首次运行下载财务数据 |

## 五、与红利周期策略的区别

| 维度 | 红利周期策略 | FCF 现金流策略 |
|:---|:---|:---|
| **选股逻辑** | 股息率 + 股债息差 + 确定性 | 自由现金流率（FCF/EV） |
| **核心指标** | 分红能力 | 现金创造能力 |
| **样本空间** | 精选 32/动态 CSI300+500 | CSI 全指 (3000+ 只) |
| **持仓数量** | ≤15 只 | 100 只（分散度极高） |
| **加权方式** | 评分 → 阶梯仓位 | FCF 加权（上限 10%） |
| **行业约束** | 弱周期/消费/周期分散 | 无行业限制（仅踢金融/地产） |
| **调仓频率** | 季度末最后一个周五 | 每季第二周五 |
| **适用场景** | 追求稳定现金流（退休/养老） | 追求高质量成长 + 价值 |

## 六、风险提示

1. **历史回测不代表未来表现**：FCF 策略在特定市场环境下可能表现不佳
2. **财务数据时效性**：年报公告有 1-4 个月滞后，调仓时使用的是"过时"数据
3. **API 数据质量**：tushare 的历史财务数据可能存在缺失或误差
4. **成交额估算**：用调仓日单日换手率×流通市值估算日均成交额，非精确年度均值
5. **行业集中风险**：FCF 率高的行业可能集中在某些周期性行业，需关注行业暴露
